# Contributing

Thanks for your interest in contributing to containr!

## Reporting bugs
- Use the bug report template and include steps to reproduce.
- Include OS, terminal, and containr version details.

## Feature requests
- Use the feature request template.
- Explain the use case and expected behavior.

## Development setup
- Rust stable is required.
- Build: `cargo build`
- Test: `cargo test`

## Pull requests
- Keep changes focused and well-scoped.
- Add tests when behavior changes.
- Update docs if user-facing behavior changes.
