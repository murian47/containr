# Codex Infos and Rules

This document captures session-level rules and conventions that should be
applied consistently.

## Versioning and Commits

- Before any commit that includes changes to source files, increment the patch
  version by 1.
- Do not increment the patch version if the minor or major version has already
  been adjusted for the same change set.
- Commit messages always summarize the changes.
- If it is unclear whether a version bump is required, ask first.

## Communication and Workflow

- When a prompt includes questions, answer them before making code changes.
- Communication within this session is in German.

## Documentation and Code Comments

- Documentation and code comments must be in English.
- Documentation and code comments must use ASCII characters only.
